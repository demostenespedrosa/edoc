DOCUMENTO TÉCNICO - E-DOC

1. Nome do Projeto:

E-Doc

2. Público Alvo:

Microempresas

3. Problema:

As microempresas enfrentam dificuldades em unificar documentos, como notas fiscais e ordens de serviço, em um formato padrão e organizado.

4. Solução:

Desenvolver o E-Doc, um sistema para unificar esses documentos em um formato eletrônico, simplificando o processo de armazenamento e acesso.

5. Tecnologias Utilizadas:

  • Linguagem de Programação: C# & HTML
  • Plataforma: Windows

6. Monetização:

Modelo baseado em suporte anual para manutenção e atualizações do sistema.

7. Requisitos Não Funcionais:

  • Usabilidade: Interface amigável e intuitiva.
  • Confiabilidade: Sistema estável e seguro.

8. Requisitos Funcionais:

  • Cadastro de Documentos: Permitir o cadastro de notas fiscais e ordens de serviço.
  • Geração de PDF: Gerar arquivos PDF de documentos cadastrados.

9. UML - Caso de Uso:

10. Requisitos de Classes:

Banco de Dados: Estrutura para armazenar dados dos documentos cadastrados.

  • Criar Documento(): Função para abrir o formulário de criação de documento.
  • Abrir TXT(): Gera o arquivo .txt com os dados inseridos.
  • Limpar dados anteriores(): Limpa os dados preenchidos.

11. Estrutura do Projeto:

  • Form1 (Tela Inicial): Tela de entrada com botão para criar documento.
  • FormDocumento (Documento Fiscal): Tela de preenchimento dos dados do documento.

12. Funcionamento do Aplicativo:

  • Abrir o aplicativo e acessar a Tela Inicial.
  • Clicar em Criar Documento para preencher os dados do documento.
  • Gerar e visualizar o arquivo de texto com os dados inseridos.
  • Salvar e converter em PDF.
  • Limpar dados anteriores, se necessário.

13. Conclusão:

O sistema E-DOC facilita a criação e organização de documentos fiscais. Com uma interface intuitiva e funcionalidades práticas, ele torna o processo mais eficiente para microempresas.
