DOCUMENTO TÉCNICO - E-DOC

1. Nome do Projeto:

E-Doc

2. Público Alvo:

Micro-empresas

3. Problema:

As microempresas enfrentam dificuldades em unificar documentos, como notas fiscais e ordens de serviço, em um formato padrão e organizado.

4. Solução:

Desenvolver um programa denominado E-Doc, que permita a unificação desses documentos em um único formato eletrônico, denominado Documento Eletrônico. Este sistema permitirá que as microempresas centralizem e organizem seus documentos de maneira mais eficiente, simplificando o processo de armazenamento e acesso.

5. Tecnologias Utilizadas:

  • Linguagem de Programação: C# & HTML
  • Plataforma: Windows

6. Monetização:

O modelo de monetização será baseado em suporte anual para manutenção e atualizações do sistema.

7. Requisitos Não Funcionais:

  • Usabilidade: O sistema deve ser simples de usar, com uma interface amigável, permitindo que microempresas utilizem o programa sem dificuldades.
  • Confiabilidade: O sistema deve ser estável, com mínima possibilidade de falhas ou perdas de dados.

8. Requisitos Funcionais:

  • Cadastro de Documentos: O sistema deve permitir o cadastro de diferentes tipos de documentos, como notas fiscais e ordens de serviço.
  • Geração de PDF: O sistema deverá gerar e salvar os documentos em formato PDF, proporcionando fácil visualização e compartilhamento.

9. UML - Caso de Uso:

Conheça já nossa UML.

10. Requisitos de Classes:

Banco de Dados: A estrutura do banco de dados deve ser capaz de armazenar dados dos documentos cadastrados, como tipo de documento, informações relacionadas, data de cadastro, etc.

Funções:

  • CadastroDocumento(): Função responsável por cadastrar documentos no banco de dados.
  • GerarPDF(): Função para gerar o arquivo PDF do documento cadastrado.
  • ConsultarDocumentos(): Função para listar documentos cadastrados.
  • ExcluirDocumento(): Função para excluir um documento do sistema.
  • GerarRelatorio(): Função para gerar relatórios dos documentos armazenados.
